# Тестирование

## Пользовательские тест-кейсы

### Case №1
**Проверка добавления файла**

Входные параметры: существует файл `tests/test_case1/file1.txt`

- Шаг 1 — запустить приложение `./file_monitoring`
- Шаг 2 — ввести команду `add tests/test_case1/file1.txt`

Результат: в консоли выводится сообщение `[ADDED]` с путём к файлу и его размером в байтах

---

### Case №2
**Проверка добавления несуществующего файла**

Входные параметры: -

- Шаг 1 — запустить приложение `./file_monitoring`
- Шаг 2 — ввести `add tests/test_case2/file2.txt`

Результат: в консоли выводится сообщение `[ERROR]`, приложение не завершается

---

### Case №3
**Проверка обнаружения изменения файла с изменением размера**

Входные параметры: существует файл `tests/test_case3/file1.txt`

- Шаг 1 — запустить приложение `./file_monitoring`
- Шаг 2 — ввести `add tests/test_case3/file1.txt`
- Шаг 3 — в отдельном редакторе дописать любой символ в `tests/test_case3/file1.txt`

Результат: в консоли появляется сообщение `[MODIFIED]` с обновлённым размером файла

---

### Case №4
**Проверка обнаружения изменения файла без изменения размера**

Входные параметры: файл `tests/test_case4/file_same_size.txt` содержит ровно один символ `1`

- Шаг 1 — запустить приложение `./file_monitoring`
- Шаг 2 — ввести `add tests/test_case4/file_same_size.txt`
- Шаг 3 — заменить символ `1` на символ `2` в `tests/test_case4/file_same_size.txt`

Результат: в консоли появляется сообщение `[MODIFIED]`, несмотря на то что размер файла не изменился

---

### Case №5
**Проверка обнаружения удаления файла**

Входные параметры: существует файл `tests/test_case5/file_recreate.txt`

- Шаг 1 — запустить приложение `./file_monitoring`
- Шаг 2 — ввести `add tests/test_case5/file_recreate.txt`
- Шаг 3 — удалить файл `tests/test_case5/file_recreate.txt`

Результат: в консоли появляется сообщение `[DELETED]` с путём к удалённому файлу

---

### Case №6
**Проверка удаления файла из наблюдения**

Входные параметры: существует файл `tests/test_case6/file1.txt`

- Шаг 1 — запустить приложение `./file_monitoring`
- Шаг 2 — ввести `add tests/test_case6/file1.txt`
- Шаг 3 — ввести `remove tests/test_case6/file1.txt`
- Шаг 4 — в отдельном редакторе изменить `tests/test_case6/file1.txt`

Результат: в консоли отсутствуют новые события по этому файлу

---

### Case №7
**Проверка рекурсивного добавления директории**

Входные параметры: директория `tests/test_case7/testdir/` содержит файл `nested1.txt`

- Шаг 1 — запустить приложение `./file_monitoring`
- Шаг 2 — ввести `add tests/test_case7/testdir`

Результат: в консоли выводится сообщение `[ADDED]` для каждого файла внутри директории

---

### Case №8
**Проверка отсутствия дублирования при добавлении файла из уже наблюдаемой директории**

Входные параметры: директория `tests/test_case8/testdir/` содержит файл `nested1.txt`

- Шаг 1 — запустить приложение `./file_monitoring`
- Шаг 2 — ввести `add tests/test_case8/testdir`
- Шаг 3 — ввести `add tests/test_case8/testdir/nested1.txt`

Результат: сообщение `[ADDED]` для `nested1.txt` появляется только один раз — на шаге 2; на шаге 3 новых сообщений нет

---

### Case №9
**Проверка удаления и повторного создания файла**

Входные параметры: существует файл `tests/test_case9/file_recreate.txt`

- Шаг 1 — запустить приложение `./file_monitoring`
- Шаг 2 — ввести `add tests/test_case9/file_recreate.txt`
- Шаг 3 — удалить файл `tests/test_case9/file_recreate.txt`
- Шаг 4 — повторно создать файл `tests/test_case9/file_recreate.txt` с любым содержимым

Результат: сначала фиксируется событие `[DELETED]`, затем после повторного создания — событие `[MODIFIED]`

---

### Case №10
**Проверка команды help**

Входные параметры: -

- Шаг 1 — запустить приложение `./file_monitoring`
- Шаг 2 — ввести команду `help`

Результат: в консоли выводится список доступных команд (`add`, `remove`, `exit`), приложение не завершается
